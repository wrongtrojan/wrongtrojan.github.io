设计基础

程序编写
#

  • 程序编写步骤

    1. 编辑：使用编辑器编写C++源码，生成.cpp源文件
    2. 编译：编译器将源文件转换为机器语言.obj目标文件,同时检查语法错误
    3. 链接:链接器将目标文件与系统库\其他模块结合,生成.exe可执行文件
    4. 调试:运行可执行文件,检查并修复错误

  • C++程序的基本单位/模块是函数

关键字
#

  • 语言本身预定义的,具有固定语义和用途的特殊标识符

  • 不是关键字的标识符

    1. 自定义标识符:自行命名的变量,函数,命名空间等

    2. 标准库相关的标识符:C++标准库中的类,函数,命名空间,宏等

      类别 示例
      命名空间 std
      容器/类型 string vector map set
      输入输出 cout cin endl cerr
      宏/类型别名 NULL size_t ssize_t
      算法/工具 sort find make_pair

    3. 编译器扩展:各编译器的扩展语法不是标准C++关键字
      (各种预编译指令如include define)

    4. 字面量,常量(true/false/nullptr除外)

  • *override/final是上下文关键字

运算符
#

  • 运算符优先级

    运算符类型 具体运算符
    基础/成员访问 () [] . -> :: ++(后置) --(后置)
    单目运算 ! ~ ++(前置) --(后置) + - &(取地址) *(取值)
    sizeof typeid new/delete 类型转换(如stati_cast)
    算术运算 * / % > + -
    位移运算 <<(左移) >>(右移)
    关系运算 < <= > >= > == !=
    按位运算 & > ^ > |
    逻辑运算 && > ||
    条件运算 ?:
    赋值运算 = (算术运算/位移运算/按位运算)=
    逗号运算 ,

  • 几类运算符特性

    1. 位移运算补零

      运算类型 操作数类型 补位规则
      左移<< 所有整数类型 一律补0
      右移>> 无符号整数 补零
      右移>> 有符号整数 补符号位

    2. 逗号运算符规则:从左往右计算表达式,返回最后一个的值

      #include <iostream>
using namespace std;
void Func(int,int){return}
int main(){
   int a=5,b=10;
   cout<<(a,b)<<endl; //输出为b,即10
   func((1,2),(3,4,5)); //调用func传入2和5两个参数
   return 0;
}

    1. 模除运算的符号同号取正异号取负 
         cout<<-3%2<<endl; //输出-1
   cout<<-3%-2<<endl; //输出-1
   cout<<3%2<<endl; //输出1
   cout<<3%-2<<endl; //输出1

    1. 逻辑运算符:||和&&有短路效应

左值与右值
#

  • 左值有持久的内存地址，右值是临时的无持久地址
  • 函数表达式func()作为左右值
    函数返回类型 func()类别 能否取地址
    左值引用(T&) 左值(lvalue) ✅
    右值引用(T&&) 亡值(xvalue,属于右值) ✅
    值类型(T) 纯右值(prvalue) ❌

数组,指针,字符串

数组
#

  • 初始化

    1. int arr[N]={n}则第一个元素被初始化为n,其余元素初始化为0

    2. 多维数组初始化,第一维可由初始化列表推断,其他维必须显式指定

      //以二维数组为例
! int a[5][]; //错误:第二维必须显式指定大小
! int a[][5]; //错误:第一维没有提供初始化列表
int a[][3]={{1,3,5},{2}}; //合法

  • 数组名

    1. 数组名本质是常量指针,不可更改本身的值 
      int arr[10]={0};
*(arr+1)=1; //合法,做指针运算但没改变arr本身值
! *++arr=1; //错误!前置++会修改arr本身的值

指针
#

  • 指针"错误"类型

    1. 悬挂指针:delete指针指向内存后未将指针置为nullptr

    2. 野指针: 指针定义时没有初始化,随机指向内存中某个位置

  • 几类指针特性

    1. void*指针不可以直接赋给(隐式转换)非void指针 
      void *void_ptr=new int(10);
! int *int_ptr=void_tpr; //错误!

    1. char*指针
      1. char*指针可以指向字符串字面量,其他类型不可以指向字面量
      2. cout默认输出char*指针指向的字符串内容,若要输出地址值需显示转换成void*指针

  • 指针与数组

    1. (int*) p[]与int *p[] 
      int arr[10];
(int*) p[10]=arr;//指向数组的指针
int *num;
int *p[10]={num};//元素是指针的数组

    1. int **ptr与int arr[N][N] 
      /*右侧返回一个指向[指针数组的首地址]的指针(指针的地址),
由二级指针接收*/
int **ptr1=new (int*)[10];

/*右侧返回一个指向[含20个元素的一维数组]的指针
由数组指针来接收*/ 
int (*ptr2)[20]=new int[10][20]

  • 指针运算的单位: 指针运算以指向类型的大小为单位

  • 指针与函数:函数不可返回局部变量指针,常使用new向堆区申请

字符串
#

  • 字符数组(C风格)

    1. 常见操作函数

      函数 功能
      strlen 字符串长度(不含\0)
      strcpy 字符串拷贝
      strcat 字符串拼接
      strcmp 字符串比较
      strchr 查找字符首次出现位置
      strstr 查找子串首次出现位置

    2. 给字符数组赋初值时末位\0需占位

      ! char s[3]="abc" //错误,末位'\0'占位需4个连续地址
char s[4]="abc" //合法

  • string类

    1. 不可用字符串字面量拼接 
      ! string s="hello"+"world"; //错误!字面量是常指针,不允许指针相加
string s=string("hello")+"world"; //合法

    1. 字符串长度计算中转义字符(如\n)只占1个字符位,末位\0不计入计算

类成员的初始化
#

  • 初始化顺序

    1. 成员的初始化顺序由类中成员声明顺序决定 
      class Example {
int x; // 声明顺序1
int y; // 声明顺序2
public:
   Example(): y(1),x(y) { 
      //实际顺序:先初始化x(y,随机值),再初始化y(值1)
      cout<<x<<" "<<y<<endl; //输出:随机值 1
   }
};

    1. 成员列表初始化在构造体赋值之前 
      class Example {
   int a;       //普通成员
   const int b; //const成员(只能初始化,不能赋值)
   int& c;      //引用成员(只能初始化,不能赋值)
public:
Example(int x, int& ref) : a(x), b(x+1), c(ref) {
      a = x*2; //覆盖a的初始值(x→x*2)
!    b = x+2; //错误!const成员已初始化,无法赋值
!    c = ref+1; //错误!引用已绑定,无法赋值
   }
};

  • 类的引用成员与常量成员应在初始化列表中完成初始化

前向引用声明
#

  • 让编译器知到"实体存在",但不知道大小,成员,实现

  • 成员的声明具有限制

    #pragma once
class ExampleB;
class ExampleC;
class ExampleA{
   ExampleB *bptr1; //声明指针
   ExampleB &bptr2; //声明引用
public:
   ExampleC FunC(ExampleC &C); //声明函数参数/返回值
};

  • 禁止的操作
    实例化该类型对象(Example E;编译器需知道大小)
    使用sizeof(Example)/alignof(Example)(编译器需知道大小)
    访问该类型成员(E.<menber>),编译器需知道成员
    继承该类型(class Example1:public Example2{};编译器需知道布局)
    调用该类型函数(编译器需知道函数实现)

复制构造函数实现深层拷贝
# 

template <typename T1,typename T2>
class Example{
   T1 data;
   T2 *ptr;
public:
   Example() : data(T1()), pr(nullptr) {}
   Example(const T1& d, const T2 & p_val) : data(d) {
        ptr = new T2(p_val); //分配内存并初始化
   }
   Example(const Example &E){
      data=E.data;
      if (E.ptr != nullptr) { //空指针判断，避免解引用空指针
            ptr = new T2(*E.ptr); //分配新内存，用源对象pr指向的内容初始化
      }else {
            ptr = nullptr;
         }
   }
   ~Example(){
      delete ptr; //释放pr指向的内存
      ptr=nullpr; //置空避免野指针
   }
};

类的继承与多态
#

  • 类的继承

    1. 继承方式

      基类成员属性 公有继承 保护继承 私有继承(默认)
      公有 公有 保护 私有
      保护 保护 保护 私有
      私有 不可见 不可见 不可见

    2. 虚继承的语法格式(解决菱形继承问题)

      class Derived : virtual public Base{};
//等价于
class Derived : public virtual Base{};

  • 类的多态

    1. 虚函数:构造函数不能是虚函数,析构函数一般是虚函数

    2. 基类指针/引用可以指向派生类对象,但只能调用从基类继承的成员

    3. 在派生类中写基类同名函数

      函数类型 绑定方式 行为 使用基类指针指派生类对象
      普通函数 静态绑定 隐藏 调用函数时依旧使用基类版本
      虚函数 动态绑定 改写 调用函数时使用派生类版本

    4. 纯虚函数与抽象类

      1. 语法形式如 virtual void func()=0
      2. 只实现部分纯虚函数的派生类依旧是抽象类
      3. 纯虚析构函数必须在类外提供函数体(定义),否则链接报错 
        class Base{
public:
   virtual ~Base()=0 //声明纯虚析构函数
}
Base::~Base(){
   //所有子类共有的清理工作
};
class Derived: public Base{
public:
   ~Derived(){
      //执行派生类的析构函数体,执行结束后会调用~Base()
   }
}

运算符重载
#

  • 只能重载为成员函数的运算符: = -> [] ()

  • 不能重载的运算符: . .* :: ?: sizeof typeid

  • 前置++与后置++
    前置++:没有参数,返回引用 Type& operator ++();

    class Example(){
    int value;
 public:
    Example(int v=0):value(v){}

    Example& operator++(){
       this->value++;
       return *this;
    }
}

    后置++:有哑元参数int,返回旧值拷贝 Type operator++(int);

    class Example(){
   int value;
public:
   Example(int v=0):value(v){}

   Example operator++(int){
      Example temp=*this;
      this->value++;
      return temp;
   }
}

  • 赋值运算=实现连续赋值m1=m2=m3:

    1. 返回值需是对象引用:m1=(m2=m3),m2=m3需返回m2的引用才能继续赋值

    2. 参数是对象的引用:避免不必要的拷贝,通常使用const M& 

       //函数原型一般为
   M& operator=(const M&);

  • 插入运算符<<

    1. 左端为输入流,且访问类私有属性,则一般声明为类友元函数

    2. 返回值类型为ostream&支持连续链式输出

    3. 函数参数为ostream&与类对象常引用,避免不必要拷贝

数据访问控制
#

static
#

  • static与类

    1. 类内static变量必须在外部定义 
      class Example{
   static int val; //仅声明不定义
}
//必须在类外定义且不能用static修饰
//注意:定义与访问是独立的规则
int Example::val=10;

    1. 静态方法不能直接访问非静态成员
      静态方法无this指针,无法直接访问非静态成员 
      class Example{
   static int val1;
   int val2=5;
public:
   static void func(){
   !  cout<<val1<<endl //合法
      cout<<val2<<endl //错误!不能直接访问非静态成员
   }
}
//必须在类外定义且不能用static修饰
int Example::val1=10;

    1. 类的静态成员函数属于整个继承体系

  • static与函数

    1. 函数中的static局部变量只在第一次调用时初始化

const
#

  • const与指针

    写法 不可修改对象
    const int *ptr
    int const *ptr    		 不可修改指向的对象(*p)
    int * const ptr 不可修改ptr的指向(ptr)
    const int* const p 指向合指向对象均不可修改

  • const引用

    1. const引用可以绑定任意对象,绑定后无法通过引用修改
    2. const引用支持隐式转换,非const引用不行 
      double d=3.14;
const int &ref1=d; //合法
! int &ref2=d; //错误!非const引用不能绑定临时对象

  • const与类

    1. const成员变量只能在构造函数初始化列表中初始化 
      class Example{
 const int val; //const成员变量
public:
! Example() { 
     val = 10; //错误!此时val已初始化,无法修改
  }
Example() : val(10) {} //仅初始化列表能初始化const成员
};

    1. const成员函数const修饰在参数列表后
      例如void func() const{return}

    2. const对象只能调用const成员函数,非const对象任意调用

  • const与函数

    1. 参数中的const引用/指针限制函数修改实参

    2. 返回值中的const引用/指针限制调用方修改原对象

  • 常量只能由常量指针来指向,由常引用来引用

友元
#

  • 友元类

    1. 在类型A中声明friend class B是A授权允许B访问A的私有成员 
      class B; //前置声明
class A{
   int valA=10;
   friend class B; //A授权B访问自己的私有成员
public:
   void accessB(B& b){
   !  std::cout<<b.valB; //错误!A不是B的友元，无法访问B的私有成员 
   }
};
class B{   
 int valB=20;
public:
   void accessA(A &a){
     std::cout <<a.valA; //合法:B是A的友元
   }
};

    1. 友元关系不可继承不可传递

  • 友元函数

    1. 友元函数在类内声明但不是该类的成员函数
      本质是授权允许函数访问类的私有成员 
      class Example{
private:
   int val = 10;
   friend void friendfunc(Example &obj); //声明友元函数全局函数
public:
   void func() {
   !  this->friendfunc(*this);// 错误!friendfunc不是成员函数，不能用this调用 
   }
};
void friendfunc(Example &obj) {
   std::cout<<obj.val;
}

    1. 将重载函数声明为友元函数,只会针对特定重载版本,而非所有
